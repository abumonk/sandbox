### Verdict table — column schema (normative)

The delta report MUST use a single markdown table with exactly this header:

```
| module | current_location | verdict | canonical_target | replaced_by_pass | source_adventures | dedup_row | notes |
```

Column semantics:

- `module` — bare module filename without `.py` (e.g. `studio_verify`).
  Must match one of the TC-012 grep tokens for verify/codegen rows.
- `current_location` — filesystem path, e.g. `ark/tools/verify/studio_verify.py`.
- `verdict` — one of exactly `KEEP` / `MERGE-INTO-HARNESS` / `RETIRE`.
  `KEEP-AS-PLUGIN` is not allowed; a domain verifier that survives as a
  registered plugin uses verdict `MERGE-INTO-HARNESS` with the plugin name
  recorded in `canonical_target`.
- `canonical_target` — the destination module/class after unification.
  Examples: `ark_verify.Harness`, `Codegen.Pipeline:target=rust`,
  `Codegen.Pipeline:target=claude-agent-md`. Empty only when `verdict=RETIRE`.
- `replaced_by_pass` — for verify rows only: comma-separated list of the
  canonical shared passes from the 4-pass spec that subsume domain checks in
  this module. Empty for codegen rows.
- `source_adventures` — adventure IDs that introduced or extended the module
  (e.g. `ADV-001,ADV-002`).
- `dedup_row` — citation of the deduplication-matrix row number(s) that map
  to this verdict. Format: `dedup:N` (see Citation Format below). May list
  multiple: `dedup:1,dedup:4`. Use `—` if no dedup row applies.
- `notes` — one-sentence rationale (≤ 140 chars).

### 4-pass canonical harness spec (normative)

Each row below defines one shared pass. The delta report MUST reproduce each
pass name, Python signature, and non-empty `replaces` list verbatim in a
`## Shared Verify Passes` section (header required; TC-011 greps for the four
names). Signatures are the public `register_pass(...)` entries.

**Pass 1 — `dag_acyclicity`**
- Signature: `dag_acyclicity(items: list, node_kind: str, edge_kind: str,
  edge_attr: str, *, terminals_ok: bool = True) -> list[Result]`
- Method: assign Z3 Int ordinal per node; for every edge A→B assert
  `ord(B) > ord(A)`; UNSAT ⇒ cycle. Subsumes every domain's hand-rolled
  ordinal-chain verifier.
- Replaces (source verifiers):
  - `studio_verify.verify_escalation_acyclicity` (role.escalates_to, ADV-003)
  - `visual_verify` review-cycle check (visual_review.replaces_screenshot,
    ADV-006)
  - `agent_verify` model-fallback acyclicity (model_config.fallback_to,
    ADV-005)
  - `graph_verify.check_no_inheritance_cycles` (inherits edges, ADV-002)

**Pass 2 — `opaque_primitive`**
- Signature: `opaque_primitive(name: str, kind: str, arg_sorts: list,
  result_sort) -> Function` (registrar; returns the Z3 uninterpreted fn and
  tags downstream obligations `PASS_OPAQUE`).
- Method: lifts today's `OPAQUE_PRIMITIVES` + `apply_opaque` from
  `ark_verify` into the shared harness; `expression_smt.PRIMITIVE_Z3`
  becomes the canonical registry.
- Replaces (source verifiers):
  - `ark_verify.OPAQUE_PRIMITIVES` table + `apply_opaque` (ADV-002)
  - `expression_smt.PRIMITIVE_Z3` wrapper (ADV-002)
  - Any ad-hoc `PASS_OPAQUE` emission inside `studio_verify`,
    `evolution_verify`, `agent_verify`, `visual_verify` (ADV-003/004/005/006
    — currently mirrored; becomes a single call site).

**Pass 3 — `numeric_interval`**
- Signature: `numeric_interval(sym: Expr, lo: Optional[Num], hi: Optional[Num],
  *, strict_lo: bool = False, strict_hi: bool = False) -> Result`
- Method: asserts `lo ≤ sym ≤ hi` (or strict) with a Z3 Real/Int check.
  Sum-to-1 constraints compose two calls (`numeric_interval(sum, 1.0, 1.0)`
  on the Sum expression plus `numeric_interval(x, 0, None)` per weight).
- Replaces (source verifiers):
  - `ark_verify` $data bounds check (ADV-001, generic)
  - `evolution_verify` split-ratio sum-to-1, fitness-weight sum-to-1,
    tolerance (0, 1.0], optimizer iterations > 0, population > 0 (ADV-004)
  - `agent_verify` cpu_cores > 0, memory_mb > 0, timeout_seconds > 0
    (ADV-005)
  - `visual_verify` width > 0, height > 0, scale > 0, annotation coords
    within viewport bounds (ADV-006)

**Pass 4 — `reference_exists`**
- Signature: `reference_exists(ref: str, target_index: dict[str, Any],
  *, ref_label: str = "") -> Result`
- Method: pure membership check (no Z3 needed) lifted to the harness so every
  domain verifier shares one error format.
- Replaces (source verifiers):
  - `studio_verify` command.required_role → role index, hook.event →
    ALLOWED_HOOK_EVENTS (ADV-003)
  - `evolution_verify` cross-ref integrity (evolution_run refs,
    evolution_target constraint refs) (ADV-004)
  - `agent_verify` gateway.agent_ref, gateway platforms, cron.agent_ref,
    cron.platform_delivery, agent.model_ref, backend refs (ADV-005)
  - `visual_verify` visual_review.target → diagram/preview/screenshot index,
    diagram.type ∈ VALID_DIAGRAM_TYPES, render format ∈ VALID_RENDER_FORMATS
    (ADV-006)
  - `graph_verify.check_no_dangling_edges` (edge source/target → nodes,
    ADV-002)

A domain verifier residue — anything NOT covered by the four passes above —
must be enumerated explicitly in the delta's `## Domain-Specific Residue`
section. Known residue (non-exhaustive):
- `studio_verify` rule-constraint SAT (Z3 satisfiability of user-written
  predicate) — stays as a plugin; not general.
- `agent_verify` skill-trigger overlap warning — pattern-matching heuristic,
  not a SAT/interval check.
- `evolution_verify` benchmark-gate tolerance semantics (domain-specific
  tolerance meaning) — stays as a plugin wrapper around `numeric_interval`.

### Domain verifier → canonical pass mapping (summary table required)

Delta report MUST include a second table titled

```
| domain_verifier | dag_acyclicity | opaque_primitive | numeric_interval | reference_exists | residue |
```

One row per domain verifier module (`studio_verify`, `evolution_verify`,
`agent_verify`, `visual_verify`, `ark_verify`, `expression_smt`,

pass, or `—`. The `residue` column lists domain-specific checks that stay as

names exist.

### Codegen classification criteria (normative)

Apply these rules in order to assign a verdict to each codegen module:

1. If the module emits **target-language source code** (Rust/C++/Proto), its
   verdict is `MERGE-INTO-HARNESS` with `canonical_target=Codegen.Pipeline:
   target=<lang>`. The domain-specific emission logic becomes an emitter
   plugin keyed on target. Applies to `ark_codegen` (rust/cpp/proto).
2. If the module emits **configuration artefacts consumed by external
   tooling** (YAML, JSON, markdown, shell, crontab, docker-compose), its
   verdict is `MERGE-INTO-HARNESS` with `canonical_target=Codegen.Pipeline:
   target=<artefact-kind>` where `<artefact-kind>` is drawn from the
   canonical emitter set `{claude-agent-md, claude-command-md, hooks-json,
   agent-yaml, cron-entry, skill-md, docker-compose, jsonl-dataset,
   python-scorer-skeleton, run-config-json, report-md, mermaid, html-preview,
   annotation-overlay-json, review-shell, catalog-index-html,
   render-config-json}`. Applies to `studio_codegen`, `agent_codegen`,
   `evolution_codegen`, `visual_codegen`.
3. If the module is a **primitive lookup table** referenced by other codegen
   modules (no file output of its own), verdict is `KEEP` with
   `canonical_target=Codegen.Pipeline.PrimitiveRegistry`. Applies to
   `expression_primitives`.
4. If the module is a **pure lookup/diff/impact analyser over the AST** with
   no per-domain code, verdict is `KEEP` unchanged. Applies to `ark_impact`
   (verify directory, but not a verifier).

No codegen module is `RETIRE` in v1. Domain-specific emission keeps 1:1 parity
with today's output; the restructure is organisational, not behavioural.

`design-deduplication-matrix.md`). Format: `dedup:N` where N is the 1-based
seed index. Multiple citations are comma-separated: `dedup:1,dedup:4`.

The four builder-bucket dedup rows that MUST be cited at least once across
the delta:

- `dedup:4` — PASS_OPAQUE Z3 pattern (ADV-002/003/004/006/008; canonical
  `verify.opaque_primitive`). Cited on every row whose `replaced_by_pass`
  contains `opaque_primitive`, and on `expression_smt`, `ark_verify`.

  surfaces in codegen via `expression_primitives` which mirrors stdlib).
  Cited on `expression_primitives` and any codegen row that emits from

- `dedup:6` — Skill definitions (ADV-003/004/005). Cited on `studio_codegen`
  (skills via roles), `agent_codegen` (skill.md emitter), and any verifier
  check that references a Skill ref.
