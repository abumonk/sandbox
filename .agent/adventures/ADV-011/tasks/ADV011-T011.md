### `run-all.sh` Structure (required)

Every point below is a hard requirement; any deviation is a design bug.

1. **Shebang**: first line must be exactly `#!/usr/bin/env bash`.
2. **Safety header**: `set -euo pipefail` on the line after the top
   comment block.
3. **ROOT resolution**: `ROOT="$(cd "$(dirname "$0")/../../../.." && pwd)"`
   followed by `cd "$ROOT"`. This makes the script runnable from any
   cwd; every proof command is written relative to repo root.
4. **`FAIL` accumulator**: initialise `FAIL=0` before the first
   `run_tc` call. `run_tc` sets `FAIL=1` on any failure but does not
   early-exit — we want the full pass/fail matrix, not first-failure
   bail, so T012's grep can see every TC line.
5. **`run_tc` helper** (verbatim from T002):
   ```bash
   run_tc() {
     local id="$1"; shift
     echo "==> $id"
     if bash -c "$*"; then
       echo "    PASS $id"
     else
       echo "    FAIL $id"
       FAIL=1
     fi
   }
   ```
   Note: `bash -c "$*"` is deliberate — the proof commands contain
   shell pipelines, `for` loops, and command substitution; they must
   be re-parsed by a fresh shell. Using `"$@"` or `eval` would break
   the verbatim-copy contract.
6. **One `run_tc` call per TC**, in manifest numeric order with
   `TC-TS-1` placed between `TC-002` and `TC-003` (alphabetic collation,
   matches T002's table ordering). The full list the implementer must
   emit (22 calls):

   TC-001, TC-002, TC-TS-1, TC-003, TC-004, TC-005, TC-006, TC-007,
   TC-008, TC-009, TC-010, TC-011, TC-012, TC-013, TC-014, TC-015,
   TC-016, TC-017, TC-018, TC-019 (self-reference: see rule 8 below),
   TC-020 (self-reference: see rule 8 below), TC-021.

7. **Verbatim copy of proof commands**. For each TC, the string passed
   to `run_tc` is the exact `Proof Command` cell from the manifest,
   single-quoted so pipes, backslashes, `\|` escapes, and embedded
   double-quotes survive. Example for TC-001:
   ```bash
   run_tc TC-001 'test -f .agent/adventures/ADV-011/research/concept-inventory.md && for a in 001 002 003 004 005 006 007 008 010; do grep -q "ADV-$a" .agent/adventures/ADV-011/research/concept-inventory.md || exit 1; done'
   ```
   If a manifest cell contains a literal single quote, escape with
   `'\''`. None of the current 22 cells contain single quotes, so no
   escaping is required — but the implementer must still scan each
   before emitting.
8. **Self-reference TCs (TC-019, TC-020)** — the aggregator cannot run
   itself recursively. For TC-019, emit:
   ```bash
   run_tc TC-019 'test -x .agent/adventures/ADV-011/tests/run-all.sh || test -f .agent/adventures/ADV-011/tests/run-all.sh'
   ```
   This degrades TC-019 inside the aggregator to a file-existence
   check; the real `bash run-all.sh` proof is executed by T012 and by
   the pipeline's CI driver (whatever invokes run-all.sh in the first
   place). For TC-020, emit the discover command directly:
   ```bash
   run_tc TC-020 'python -m unittest discover -s .agent/adventures/ADV-011/tests -v'
   ```
   TC-016 is already executed via unittest discover, so it is also
   covered transitively, but we still emit an explicit `run_tc TC-016`
   line to keep the grep-friendly output consistent across all 22 TCs.
9. **Terminal line**: `exit "$FAIL"`. No other `exit` statements.
10. **Output format contract with T012**. Every TC produces exactly
    three lines on stdout:
    ```
    ==> TC-xxx
        PASS TC-xxx
    ```
    (or `FAIL TC-xxx`). T012's final-validation-report greps these
    `PASS TC-` / `FAIL TC-` lines to populate its summary table, so
    the exact leading-whitespace and single-space separators must be
    preserved from the `run_tc` helper above.

### `test_coverage_arithmetic.py` Structure (required)

Copy verbatim from T002 Design, with these invariants:

- **Module docstring** names TC-016: first line
  `"""Validates TC-016 — validation-coverage arithmetic.`
- **Class**: `class TestCoverageArithmetic(unittest.TestCase):` with
  class docstring `"""TC-016: covered + retired + deferred == total_source_TCs."""`.
- **`setUp`**: reads
  `.agent/adventures/ADV-011/research/validation-coverage.md`, stores
  text on `self.text`, asserts existence with a helpful message
  naming the upstream task (T009).
- **Helper**: `_count_rows_by_verdict` — iterates
  `self.text.splitlines()`, counts lines that start with `| TC-`,
  tallies per-verdict occurrences against the constant
  `VERDICTS = ("COVERED-BY", "RETIRED-BY", "DEFERRED-TO")`, returns
  `(total, counts_dict)`.
- **Test method 1**: `test_each_tc_has_exactly_one_verdict` with
  docstring `"""Validates TC-016 row-level integrity: every TC row matches exactly one verdict."""`.
  Asserts `total > 0` and `sum(counts.values()) == total`.
- **Test method 2**: `test_arithmetic_holds` with docstring
  `"""Validates TC-016: covered + retired + deferred == total."""`.
  Asserts
  `counts["COVERED-BY"] + counts["RETIRED-BY"] + counts["DEFERRED-TO"] == total`.
- **No `if __name__ == "__main__"` business logic** other than
  `unittest.main()`.
- **Paths**: use `pathlib.Path(".agent/adventures/ADV-011/research/validation-coverage.md")`.
  `run-all.sh` guarantees cwd == repo root, and the manifest's TC-016
  proof command is also invoked from repo root, so the relative path
  is safe.

### `test_mapping_completeness.py` Structure (required)

Copy verbatim from T002 Design, with these invariants:

- **Module docstring**: first line
  `"""Validates TC-004 (poc) and cross-file completeness.`
- **Class**: `class TestMappingCompleteness(unittest.TestCase):`.
- **`setUp`**: asserts both `concept-inventory.md` and
  `concept-mapping.md` exist; stores their texts on `self`.
- **Helper** at module scope:
  `_table_rows(text)` — generator yielding `list[str]` cell-lists for
  each pipe-table data row; skips header row (first cell in
  `{"concept", "tc"}`, case-insensitive) and skips the `|---|---|`
  separator row (regex `^\|[\s-]+\|`).
- **Test method 1**: `test_buckets_allowlist` with docstring
  `"""Validates TC-004: every mapping row's bucket is in the allowlist."""`.
  Iterates rows of `self.mapping_text`; for each row with
  `len(cells) > bucket_idx`, asserts `cells[bucket_idx].lower() in ALLOWED_BUCKETS`
  where `ALLOWED_BUCKETS = {"descriptor", "builder", "controller", "out-of-scope"}`.
  `bucket_idx = 1` per the
  `concept | bucket | source_adventure | rationale` schema
  documented in `ADV-011/schemas/entities.md#ConceptMappingRow`; if
  T003 reordered columns the test will fail loudly and point at the
  schema file.
- **Test method 2**: `test_every_inventory_concept_is_mapped` with
  docstring
  `"""Validates cross-file completeness: no concept from inventory is silently dropped."""`.
  Collects concept names from inventory column 0 into a `set`; asserts
  every name appears at least once as a case-insensitive substring of
  `self.mapping_text`. Failure message lists the first 5 missing
  concepts.

needs to reroll.

### Error Handling and Output Format (consumed by T012)

`run-all.sh` is both a test runner and a report source for T012. The
contract:

1. Exit code is `0` if and only if every `run_tc` call succeeded.
2. stdout contains exactly three lines per TC (`==>`, indented
   `PASS`/`FAIL`), emitted in the order defined above.
3. Lines are not prefixed with timestamps or colour codes — keep
   output plain ASCII so T012 can use
   `grep -c "^    PASS TC-"` and
   `grep -E "^    FAIL TC-" run-all.log` without escaping.
4. No stderr output on success. On failure, the failing proof
   command's own stderr passes through untouched — helpful for
   debugging but T012's report relies on stdout only.
5. `set -euo pipefail` is active globally, but the `run_tc` helper
   uses `if bash -c "$*"; then ... else ... fi` which is an allowed
   idiom: the `if` condition protects the enclosing script from the
   `-e` abort, so FAIL accumulation works.

### Testing Strategy

- **TC-020** is the primary unittest gate. Run
  `python -m unittest discover -s .agent/adventures/ADV-011/tests -v`
  — expect at least 4 tests discovered (2 in each unittest file),
  expect all to pass.
- **TC-019** is the aggregator gate. Run
  `bash .agent/adventures/ADV-011/tests/run-all.sh` — expect exit 0
  and `PASS TC-` on every line. Any `FAIL TC-xxx` line identifies a
  broken upstream artefact and requires rerolling that task, not
  editing the test.
- **Self-check on the docstring convention**:
  `grep -cE '"""Validates TC-' tests/test_*.py` ≥ 4.
- **Output-format check** (simulates T012's consumption):
  `bash run-all.sh | grep -cE "^    (PASS|FAIL) TC-"` must equal 22.

### Risks

- **Proof-command drift** — if T002's `test-strategy.md` has any
  character difference from the manifest's proof column, `run-all.sh`
  may pass strategy checks but fail manifest checks (or vice versa).
  Mitigation: copy proof commands from the **manifest**, not from
  `test-strategy.md`; T002's design guarantees they agree, but the
  manifest is the authoritative source.
- **Windows line endings / executable bit** — Git Bash on Windows may
  write CRLF line endings on `.sh` files, which breaks `#!/usr/bin/env
  bash`. Mitigation: ensure the editor writes LF; if necessary add a
  `.gitattributes` entry `*.sh text eol=lf` (out of scope for this
  task, but call it out if CI fails mysteriously).
- **Python discovery path** — `unittest discover` requires `__init__.py`
  to treat `tests/` as a package when the dotted module form
  `.agent.adventures.ADV-011.tests.test_coverage_arithmetic` is used
  (per manifest TC-016). The dotted form actually fails because
  `ADV-011` contains a hyphen (not a valid Python identifier).
  Mitigation: `run-all.sh` uses
  `python -m unittest discover -s <path> -p <pattern>` form, which
  does not require the hyphenated path to be importable as a dotted
  module. The manifest's TC-016 proof command using the dotted form
  is therefore a latent bug; `run-all.sh` emits the discover form
  instead and flags the divergence in the commit message. T012 may
  file a follow-up to patch the manifest cell; that is out of this
  task's scope.
- **Upstream artefacts missing at dispatch time** — pipeline lead
  must verify all `depends_on` tasks are `done` before dispatching
  T011 to the researcher agent. If dispatch races a late upstream
  write, `run-all.sh` fails loudly and the researcher reports
  upstream-incomplete in the Log and pauses.
- **TC-019 self-reference loop** — handled by Rule 8 above
  (file-existence substitute inside the aggregator). Without this
  fix, running `bash run-all.sh` from inside a `run_tc` call would
  recurse and either stack-overflow or spuriously double-count TCs.
