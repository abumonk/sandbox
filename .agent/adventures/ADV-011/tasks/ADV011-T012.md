#### `## Run-All Output`
Researcher executes `bash .agent/adventures/ADV-011/tests/run-all.sh` exactly once and pastes the complete stdout into a fenced code block labelled `text`. Format:

````
## Run-All Output

Command: `bash .agent/adventures/ADV-011/tests/run-all.sh`
Executed: {ISO-8601 timestamp}
Exit code: 0

```text
{literal paste of run-all.sh stdout, unedited}
```
````

The fenced block MUST contain the literal string `exit 0` or `PASS` (whatever run-all.sh emits on success) so a reader can grep the report itself and confirm the green state. If run-all.sh emits no summary line, the researcher appends `Exit code: 0` as a line outside the code fence (as shown above) — this satisfies TC-021's implicit "copy-paste of actual output" acceptance criterion.

If the run fails (non-zero exit), the researcher MUST NOT proceed to write the Ready-for-Review Statement; instead they flip task status to `blocked` and log the failure. The ready statement is contingent on a green run.

This sentence is the handoff contract — the adventure orchestrator / reviewer will grep for it.

### Testing Strategy
- TC-021 proof command (shown in manifest line 96) is the authoritative check — runs nine greps on the report.
- Secondary manual check: the embedded run-all.sh output in the report must match a fresh run. Reviewer may re-run run-all.sh and diff; if the fenced paste diverges, the report is stale.
- No new tests needed — T011 already implements run-all.sh and test_coverage_arithmetic.py.

### Risks
- **Stale counts**: if the researcher writes the report before T011's tests settle, numbers drift. Mitigation: T012 depends on T011 via `depends_on`.
- **run-all.sh non-deterministic output**: if the script prints timestamps, the embedded paste will always differ from future runs. Accepted risk — the fenced block is a witness, not a hash.
- **Keyword false negatives**: if the researcher paraphrases (e.g., writes "the dedup matrix" instead of "deduplication-matrix.md"), TC-021 fails. Mitigation: the Counts table uses exact filenames — design enforces this.
- **Premature ready statement**: if run-all.sh fails, writing the ready sentence would be a false signal. Design explicitly gates the sentence on exit 0.
