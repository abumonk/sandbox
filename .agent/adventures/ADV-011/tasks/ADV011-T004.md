## Design

### Summary
Author `research/deduplication-matrix.md` as a single markdown document comprising (a) an intro paragraph citing `design-deduplication-matrix.md` and `concept-mapping.md` as inputs, (b) a master table keyed on canonical concept, (c) one row per duplicated concept (≥ 6 seeded), (d) a trailing `## Not Duplicates` section justifying every multi-source mapping row that was deliberately excluded. Do not modify `concept-mapping.md` or any other file.

Authoritative design reference: `.agent/adventures/ADV-011/designs/design-deduplication-matrix.md`.
Schema reference: `.agent/adventures/ADV-011/schemas/entities.md#DeduplicationMatrixRow`.

### Column schema (six columns, exact header row)
The table MUST have exactly the following header, column order preserved so that TC-006's awk `-F'\|'` column-count check resolves `$4` to `canonical_form`:

```
| concept | sources | canonical_form | assigned_bucket | unification_action | notes |
|---------|---------|----------------|-----------------|--------------------|-------|
```

Field semantics (per `entities.md` `DeduplicationMatrixRow`):
- `concept` — canonical concept label (short noun phrase, keyword-bearing so TC-005 grep hits the row).
- `sources` — bracketed list of `(ADV-00X: local_name)` pairs separated by `;`. At least two sources per duplication row.
- `canonical_form` — MUST be non-empty. Prefer a concrete artifact: function signature (`verify.dag_acyclicity(node_kind, edge_kind)`), struct name (`Skill`), module id (`tools/telemetry/capture.py`), or Z3 pass id (`PASS_OPAQUE`).
- `assigned_bucket` — exactly one of `descriptor | builder | controller` (mirrors the row's bucket in `concept-mapping.md`; no `out-of-scope` rows in the matrix).
- `unification_action` — MUST be non-empty. Imperative verb phrase describing the downstream transform (e.g. "promote to stdlib primitive", "hoist into shared verifier plugin", "collapse two Z3 functions into one", "document as recurring practice in controller design").
- `notes` — free text, may be empty; links to decisions.md anchors or pattern IDs when relevant.

Cell quoting rule: every cell is inline markdown on a single line; pipes inside cells must be escaped `\|`; multi-item sources use `;` as the intra-cell separator.

### Scan procedure over concept-mapping.md
After seeding, the synthesist MUST:

1. Parse `research/concept-mapping.md`'s master table. Any row whose `source_adventure` cell lists two or more `ADV-0XX` ids is a *multi-source concept*.
2. For each multi-source concept, either:
   - (a) merge it into an existing seed row if semantically identical (extend `sources` cell), or
   - (b) add a new matrix row with all six columns populated, or
   - (c) record it under `## Not Duplicates` with a one-sentence justification explaining why the concept is legitimately distinct (e.g. homonyms, different abstraction levels, distinct lifetimes).
3. Each of options (a), (b), (c) MUST leave an audit trail: the concept name must appear either in the matrix (as concept or in sources) or in `## Not Duplicates`. Nothing multi-source may silently vanish.
4. Additional likely rows to examine per design doc: `role` vs `agent`/`persona` (ADV-003 vs ADV-005), `hook` (ADV-003 vs ADV-010 `SubagentStop`), `review cycle` (ADV-006 `visual_review` vs ADV-004 human-in-loop PR gate). Classify each as row or Not-Duplicate with reasoning.

### `## Not Duplicates` section format
Trailing section titled exactly `## Not Duplicates`. Bullet list; each entry:

```
- **{concept}** — sources: (ADV-0XX: …); (ADV-0YY: …). Rationale: {≥ 1 sentence explaining why distinct}.
```

This section may be empty if every multi-source concept collapses into a matrix row, in which case a single line `- (none — all multi-source concepts absorbed into the matrix above)` is acceptable.

### Invariants for target conditions

**TC-005 (keyword grep)** — proof command greps case-insensitively for each of `Z3 ordinals`, `Lark`, `Pest`, `telemetry`, `PASS_OPAQUE`, `dogfood`, `Skill`. Invariants to preserve:
- The literal substring `Z3 ordinals` appears in the row-1 `concept` cell.
- Both `Lark` and `Pest` literal substrings appear in the row-2 `canonical_form` or `notes` cell.
- The literal substring `telemetry` appears in the row-3 `concept` cell.
- The literal substring `PASS_OPAQUE` appears in the row-4 `canonical_form` cell.
- The literal substring `dogfood` (or `dogfooding`) appears in the row-5 `concept` cell.
- The literal substring `Skill` appears in the row-6 `concept` cell.
- Do not replace these tokens with synonyms in headings or text — the grep is substring-based and case-insensitive, but authors must not rely solely on casing variants; preserve the exact tokens somewhere in the row.

**TC-006 (awk column count)** — proof command:
```
grep -E "^\|.*\|.*\|.*\|.*\|.*\|" dedup-matrix | grep -vE "^\| *canonical_form \|" | grep -vE "^\|-" | awk -F'\|' 'NF>=6 && $4 !~ /^ *$/ {c++} END {exit (c>=6)?0:1}'
```
Invariants:
- Header row begins with `| canonical_form |` pattern to be filtered out (column 3 is `canonical_form`, so header contains ` canonical_form `; the grep `-vE "^\| *canonical_form \|"` is intended to exclude the header — writers must keep `canonical_form` as the visible third header cell so this filter engages). If the grep does not match (columns are in the specified order), it is still fine: the data-row count will then include the header plus 6 seeds = 7, and the `c>=6` check still passes — but the cleanest compliance is achieved by matching the designed column order.
- Separator row begins with `|-` and is filtered by `grep -vE "^\|-"`.
- Data rows have at least 6 pipe-delimited fields (awk counts `NF >= 6`; since the row is `| a | b | c | d | e | f |`, awk's split yields 7 fields including the empty leading/trailing — NF is ≥ 7, always ≥ 6).
- Every data row's 3rd awk field (`$4` after the empty `$1`) — the `canonical_form` cell — must be non-whitespace. Writers enforce this by never leaving `canonical_form` blank.
- At least 6 data rows satisfy both conditions → `c >= 6` → exit 0.

Author-side self-check (not required by autotest, but recommended): run the awk command locally after writing; confirm exit status 0.

Mitigation: the dependency (`depends_on: [ADV011-T003]`) is enforced; reviewer must reject T004 if the mapping is absent or incomplete.
