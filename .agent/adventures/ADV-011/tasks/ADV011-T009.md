## Description
Extract every TC from ADV-001..008 manifests (via grep). Produce `research/validation-coverage.md` with one row per TC assigning `COVERED-BY | RETIRED-BY | DEFERRED-TO` per `design-validation-against-tcs.md`. Each `COVERED-BY` cites a section anchor in one of the unified designs. Each `RETIRED-BY` cites a pruning-catalog row. Each `DEFERRED-TO` cites a downstream adventure id (will be resolved by T010 producing the plan). Also produce `research/validation-report.md` summarising counts and any open gaps.

## Acceptance Criteria
- Both files `research/validation-coverage.md` and `research/validation-report.md` exist.
- Coverage matrix row count equals 278 (= 30+28+35+46+44+37+34+24 across ADV-001..008), re-grepped at T011 test time from the harvest sources named in the design.
- Every row has a non-blank verdict; verdict cell matches `^(COVERED-BY|RETIRED-BY|DEFERRED-TO)$` (TC-015).
- `covered + retired + deferred == total` arithmetic invariant holds; enforced by T011's `tests/test_coverage_arithmetic.py` (TC-016).
- Every `COVERED-BY` row cites a delta-file anchor (`descriptor-delta.md|builder-delta.md|controller-delta.md`) whose H2/H3 slug exists in that file.
- Every `RETIRED-BY` row cites `pruning-catalog.md row <n>` with `<n>` inside the catalog's data-row range.
- Every `DEFERRED-TO` row cites a downstream adventure id (`ADV-DU | ADV-BC | ADV-CC | ADV-OP | ADV-CE | ADV-UI`) from `design-downstream-adventure-plan.md`.
- `validation-report.md` contains the five required sections (Summary / Per-verdict breakdown / Per-source-adventure breakdown / Per-downstream-adventure deferral breakdown / Open gaps / Methodology) with the fixed table shapes specified in the design.

## Design

See `.agent/designs/ADV011-T009-design.md` for the full design.

**Summary.** Harvest every TC from ADV-001..008 with the regex `^\| TC-\d+ \|` against each adventure's manifest (for ADV-001, ADV-004..008) or `tests/test-strategy.md` (for ADV-002 and ADV-003, whose manifest TC tables are empty). Expected total: **278 TCs** (ADV-001: 30, ADV-002: 28, ADV-003: 35, ADV-004: 46, ADV-005: 44, ADV-006: 37, ADV-007: 34, ADV-008: 24).

Emit one row per TC in `research/validation-coverage.md` using the fixed schema `| tc_id | source_adventure | source_description | verdict | citation |`, prefixing `tc_id` with its source adventure (e.g. `ADV-001 TC-005`) to disambiguate collisions across adventures. Assign exactly one verdict per row, cited per verdict syntax:

- `COVERED-BY` → `<descriptor|builder|controller>-delta.md#<section-anchor>` with an H2/H3 slug that exists in the cited file.
- `RETIRED-BY` → `pruning-catalog.md row <n>` pointing at a data row in `research/pruning-catalog.md`.
- `DEFERRED-TO` → a downstream adventure id from `design-downstream-adventure-plan.md` (canonical seeds: `ADV-DU`, `ADV-BC`, `ADV-CC`, `ADV-OP`, optional `ADV-CE`, `ADV-UI`). T010 must honour whichever ids T009 used.

Emit `research/validation-report.md` with the fixed structure: Summary (totals + arithmetic equation printed as a literal line `covered + retired + deferred = total  →  N + N + N = 278  [PASS|FAIL]`), Per-verdict breakdown table, Per-source-adventure breakdown table (one row per ADV-001..008 plus totals), Per-downstream-adventure deferral breakdown table, Open gaps (or literal `None.`), Methodology.

Coverage arithmetic invariant (T011 `test_coverage_arithmetic.py` enforces): `covered + retired + deferred == total_source_TCs`, where the right-hand side is computed by re-grepping the harvest sources at test time. See the design doc for the full six-assertion list T011 is to implement.
