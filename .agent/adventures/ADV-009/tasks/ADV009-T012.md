## Description
Implement the three test files declared in `tests/test-strategy.md`. Tier 1 (`test_server.py`) covers backend TCs (summary block, documents endpoint, next-action logic, stdlib-only). Tier 2 (`test_ui_layout.py`) parses `index.html` statically for tab structure, CSS class presence, and `data-testid` hooks. Tier 3 (`test_ui_smoke.py`) uses Playwright if available; otherwise `unittest.skip`. Run the full discover command and verify exit 0.

## Acceptance Criteria
- [ ] `python -m unittest discover -s .agent/adventures/ADV-009/tests -p "test_*.py"` exits 0.
- [ ] Every autotest TC declared in test-strategy.md has a matching `test_` function name.
- [ ] Tier 3 tests skip cleanly if Playwright is not installed (the run still exits 0).
- [ ] Tests use only stdlib for Tier 1 and Tier 2.

### Summary

Three stdlib `unittest` modules plus a shared fixtures module. Class
and method names are pre-bound by the manifest's `Proof Command`
column, so this task is a faithful map from manifest commands to
file structure rather than a free design exercise.

### Target files
1. `.agent/adventures/ADV-009/tests/__init__.py` — empty marker for `unittest discover`.
2. `.agent/adventures/ADV-009/tests/_fixtures.py` — `make_synthetic_adventure(tmpdir, ...)` builds a minimal `.agent/adventures/ADV-TEST/` tree (manifest, permissions, 1 task, 1 design, 1 plan with `## Wave 1`/`## Wave 2`, 1 research, 1 review, adventure.log). `ServerThread(adv_root)` starts `ThreadingHTTPServer` on port 0, monkey-patches `server.ADVENTURES_DIR`/`AGENT_DIR`. `server.py` is loaded via `importlib.util.spec_from_file_location` because the package dir contains a hyphen.
3. `.agent/adventures/ADV-009/tests/test_server.py` — Tier 1. Classes: `TestSummary`, `TestNextAction`, `TestStdlibOnly` (uses `ast` + `sys.stdlib_module_names`), `TestDocumentsEndpoint` (test_types, test_waves, test_design_one_liner), `TestKnowledgePayload`, `TestStrategyDoc` (test_tc_mapping, test_framework). Maps to TC-025, TC-026, TC-027, TC-028, TC-029, TC-030, TC-034, TC-035.
4. `.agent/adventures/ADV-009/tests/test_ui_layout.py` — Tier 2. Stdlib `html.parser.HTMLParser` subclass `TagSoup` records (tag, attrs, text). Classes: `TestAuditPresence`, `TestAuditVerdicts`, `TestAuditCoverage` (read `research/audit.md`); `TestTabBar` (test_four_tabs, test_no_legacy_tabs); `TestHeader`; `TestTasksTab` (test_buckets, test_card_shape); `TestDocumentsTab` (test_chips); `TestOverview` (test_progress_bar); `TestDecisions` (test_three_cards); `TestVisualSystem` (test_classes, test_no_external, test_card_usage). Maps to TC-001..TC-005, TC-008, TC-009, TC-013, TC-018, TC-022, TC-031, TC-032, TC-033.
5. `.agent/adventures/ADV-009/tests/test_ui_smoke.py` — Tier 3. Module-level `try: import playwright` then every class decorated with `@unittest.skipUnless(PW_AVAILABLE, ...)`. `setUpClass` raises `SkipTest` if browser binary missing. Classes: `TestTaskDetail` (test_structured TC-010, test_disclosure TC-011); `TestDocuments` (test_chip_filter TC-014, test_design_header TC-015, test_plan_waves TC-016, test_review_strip TC-017); `TestOverview` (test_next_action TC-020); `TestDecisions` (test_state_post TC-024). Each behavioral TC is also covered statically in Tier 2 so coverage isn't lost when Playwright is unavailable.

### Implementation steps
1. Write `__init__.py` + `_fixtures.py` (fixture builder, `ServerThread` context manager, `http_get`/`http_post` urllib helpers, `load_console_server()` importlib loader).
2. Implement `test_server.py` mapping every method per the table above; `TestStrategyDoc` skips its class if `tests/test-strategy.md` is absent (T001 may lag).
3. Implement `test_ui_layout.py` with `TagSoup` helper and `find_by_testid` / `count_by_class` / `has_link_to`. Locate the index.html at `.agent/adventure-console/index.html`.
4. Implement `test_ui_smoke.py` with the Playwright skip guard and per-class `setUpClass` browser launch / `tearDownClass` close. Use port-0 `ServerThread`.
5. Run `python -m unittest discover -s .agent/adventures/ADV-009/tests -p "test_*.py"` from repo root; expect exit 0 (all skipped Tier-3 still counts as success).
6. Spot-check by listing test names: `python -m unittest discover -s .agent/adventures/ADV-009/tests -v 2>&1 | grep test_` to verify every autotest TC has a matching method name.
