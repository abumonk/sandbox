## Description
Add the Pipeline tab to the v2 console:
- Insert the fifth `<button>` in the tab bar.
- Add a dispatcher case and author `renderPipeline(adv)`.
- Load cytoscape (and dagre-layout if adopted) via CDN `<script>` tags.
- Mount the graph on `<div id="pipeline-canvas">`.
- Poll `/api/adventures/{id}/graph` every 5 s (default; overridable via `?poll=N`).
- Apply status-colour mapping per design.
- Render tooltips from the backend `explanations` map.
- Include a small fixed legend.

## Acceptance Criteria
- [ ] Pipeline appears as the 4th top-level tab (between Documents and Decisions) — total of 5 tabs.
- [ ] cytoscape loaded from a CDN URL (`https://cdn.jsdelivr.net/.../cytoscape...`); no local copy, no bundler artifact.
- [ ] Graph renders adventure, phase, task, document, tc, decision nodes with the status-colour mapping defined in `design-pipeline-graph-view.md` for ADV-007 sample data (task done → `--ok`, failed → `--err`, pending → `--muted`, in_progress → `--accent`; decision pending → `--warn`; document → `--text`).
- [ ] Polling uses `setTimeout` at 5000 ms default, overridable via `?poll=N` URL query; no `WebSocket` / `EventSource` references anywhere in `index.html` (grep-verifiable).
- [ ] Tooltip text is read exclusively from `explanations[id]` returned by `/api/adventures/{id}/graph`; no hardcoded tooltip strings inside `renderPipeline` / `showTooltip`.
- [ ] Polling timer is cleared on tab switch, adventure switch, and detail refresh (no orphaned pollers).

## Design

Full design: `.agent/designs/ADV009-T019-design.md`.

**Summary.** Single-file edit to `.agent/adventure-console/index.html`:
add two CDN `<script>` tags (`cytoscape@3.28.1` + optional
`cytoscape-dagre@2.5.0`/`dagre@0.8.5`) next to the existing
`marked.min.js` tag; extend the `<style>` block with `.pipeline-wrap`,
`#pipeline-canvas`, `.pipeline-legend`, `.pipeline-tooltip` rules;
insert `['pipeline', 'Pipeline']` into the tabs array so the v2 order
becomes `Overview · Tasks · Documents · Pipeline · Decisions`; add a
dispatcher branch in `renderTab`; author `renderPipeline(a)`,
`mountCytoscape(el, a)`, `pollGraph(advId)`, `applyGraphDiff(cy, g)`,
`nodeStyleFor(kind, status)`, `showTooltip/hideTooltip`,
`pipelinePollMs()`, and `stopPolling()` on the `App` object.

Status-colour mapping pulls hex values once at mount via
`getComputedStyle(document.documentElement).getPropertyValue('--ok')`
etc. (no new CSS tokens). Tooltip text is rendered only from
`this.lastGraphExplanations[node.id()]`. `applyGraphDiff` is idempotent
— it patches `data('status')` on survivors so polling does not
re-layout every 5 s. `stopPolling()` is invoked from `switchTab`,
`refreshDetail`, and the adventure-switch path in `refreshList` to
prevent orphaned timers.

If `cytoscape-dagre` fails to load, the `cy.use(cytoscapeDagre)` call
is guarded and cytoscape's built-in `breadthfirst` is used. If T018's
`/graph` endpoint is absent at implementation time, the canvas renders
a clear empty-state message rather than throwing.
