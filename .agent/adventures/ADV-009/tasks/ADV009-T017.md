## Design

**Summary.** Run Ark's unchanged verifier on each of the three verifier specs authored in ADV009-T015 and resolve each pass to one of: CLEAN (verify exits 0), REFINED (spec-side bug — edit the `.ark` file under `adventure_pipeline/specs/verify/`, never `ark/`), or DEFERRED (modelling gap — document in `adventure_pipeline/README.md` § "Deferred invariants"). Both CLEAN and DEFERRED satisfy TC-045; this task is explicitly optional and a full-deferral outcome is a valid pass.

**Primary design doc:** `.agent/adventures/ADV-009/designs/design-ark-pipeline-spec.md` (§ 5 "Optional verifier passes").
**Addendum (this task):** `.agent/designs/ADV009-T017-design.md` — concrete disposition tree, deferral template, risk notes on process-block verifier support.

**Concrete steps (condensed from addendum):**
1. Preflight: `git diff --name-only -- ark/` must be empty; record HEAD SHA.
2. Run `python ark/ark.py verify` against each of the three specs; classify CLEAN / SPEC_BUG / MODELLING_GAP.
3. For SPEC_BUG: edit the `.ark` verifier spec only (never `ark/`); re-run.
4. For MODELLING_GAP: append a bullet to `adventure_pipeline/README.md` § "Deferred invariants" using the template { spec-filename, invariant name, rationale, follow-up }; optionally weaken the affected `verify` block to a comment referencing the deferral slug.
5. Re-run all three verify commands end-to-end; capture output.
6. `git diff --exit-code -- ark/` must exit 0.
7. Append task log with per-spec disposition (CLEAN / REFINED / DEFERRED).

**Priority: deferrable** — skipping does not fail the adventure.
