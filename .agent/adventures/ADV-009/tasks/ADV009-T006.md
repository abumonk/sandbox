## Description
Replace the 9-tab bar with the four tabs Overview / Tasks / Documents / Decisions. Rebuild the header (ID, title, state pill, TC progress indicator, primary-action button from `summary.next_action`). Rebuild the sidebar row to show state-badge + ID + title + one-line subtitle. Route unrecognized legacy tab keys to Overview. Add `data-testid` hooks per test-strategy.

## Acceptance Criteria
- [ ] Exactly four top-level tabs render with `data-testid` values `tab-overview`, `tab-tasks`, `tab-documents`, `tab-decisions`.
- [ ] The Log / Knowledge / Permissions / Designs / Plans / Research / Reviews tabs are gone from the nav (no `data-testid="tab-{log|knowledge|permissions|designs|plans|research|reviews}"` anywhere in the emitted DOM or source).
- [ ] Header contains, in order: adventure ID, title, state pill, TC progress bar, primary CTA, each with the `data-testid` values listed in the design (`header-id`, `header-title`, `header-state-pill`, `tc-progress-bar`, `header-cta`).
- [ ] Primary CTA only renders when `summary.next_action` exists and `kind !== 'none'`; clicking it routes to the correct tab (Decisions for approve/blocker/promote, Tasks for `open_plan`).
- [ ] Sidebar rows do not show task counts, file counts, or raw paths; they show state badge + ID + title + one-line subtitle derived from concept/subtitle (≤80 chars, ellipsized).
- [ ] `switchTab(key)` falls back to `'overview'` for any key not in `['overview','tasks','documents','decisions']`, and the active-class toggle uses `data-testid` comparison (not text-prefix).
- [ ] Page still loads without runtime errors for an adventure whose `summary` field is missing (defensive fallback to counting `target_conditions`).

## Design
Summary: Restructure `App.render()`, `App.switchTab()`, `App.renderTab()`, and `App.refreshList()` in `.agent/adventure-console/index.html` to the v2 IA contract (four tabs, header strip, compact sidebar row). Pane bodies for Documents/Decisions land as stubs — T007/T008/T009/T010 replace them. Legacy renderers remain in the source and are retired in T011. Adds `data-testid` hooks per the test-strategy contract and a tiny set of CSS helpers (`.header-strip`, `.progress-inline`, `.cta-primary`, `.adv-subtitle`) that don't collide with T005's visual primitives.

See `.agent/designs/ADV009-T006-design.md` for full details including per-step DOM snippets, `next_action`→tab routing table, CSS additions, and Tier-2/Tier-3 test hooks.
