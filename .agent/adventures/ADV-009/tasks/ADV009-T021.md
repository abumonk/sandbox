## Description
Extend the ADV-009 test suite with:

- `tests/test_ir.py` — IR round-trip on ADV-007 + ADV-008, entity
  shape, enum serialization, orphan-id detection.
- `tests/test_graph_endpoint.py` — `/graph` payload schema test,
  node/edge cardinality smoke tests, `depends_on` POST round-trip
  including 400 cases (self-cycle, cycle-creating, unknown task).
- `tests/test_pipeline_tab.py` — static inspection of `index.html`:
  cytoscape CDN present, 5-tab tab bar, no WebSocket/EventSource usage,
  no hardcoded tooltip strings (via a pattern check).

All use stdlib `unittest`. Full discover must exit 0.

## Acceptance Criteria
- [ ] `python -m unittest discover -s .agent/adventures/ADV-009/tests -p "test_*.py"` exits 0.
- [ ] Every autotest TC from TC-039..TC-061 (excluding TC-045, TC-057 which are `proof_method: poc`) has a matching `test_` function.
- [ ] Tests use stdlib only (verified via `ast.parse` walk; only imports from `sys.stdlib_module_names` and `adventure_pipeline.*`).

## Design

See `.agent/designs/ADV009-T021-design.md` for the full design.

Summary:
- Three new stdlib-`unittest` files covering the 21 autotest TCs in the
  TC-039..TC-061 range (TC-045 and TC-057 are POC, not in scope).
- A shared `_fixtures.py` helper (not picked up by discovery) providing
  `repo_root()`, `ensure_on_syspath()`, a `TempAdventure` context
  manager, and a `ServerProcess` that boots `server.py`'s
  `ThreadingHTTPServer` on a random port.

### Files

- `.agent/adventures/ADV-009/tests/test_ir.py` — `TestSpecShapes`
  (TC-039..TC-041 via static regex over `adventure.ark`, `pipeline.ark`,
  `entities.ark`), `TestRoundTrip` (TC-042..TC-044 via
  `extract_ir("ADV-007")`, `extract_ir("ADV-008")`, manifest `tasks:`
  comparison), `TestIrEntityShape` (JSON-serializability + enum
  serialization).
- `.agent/adventures/ADV-009/tests/test_graph_endpoint.py` — boots
  `server.py` in-process. `TestImport` (TC-058 static source grep),
  `TestStdlibOnly` (TC-060 via `ast` import walk), `TestGraphShape`
  (TC-046, TC-059 payload schema), `TestDependsOn` (TC-052 happy path,
  TC-053 self-cycle, TC-054 cycle, unknown-task rejection) against a
  `TempAdventure` with 2 synthetic tasks, `TestCycleFree` (TC-061 via
  direct helper import).
- `.agent/adventures/ADV-009/tests/test_pipeline_tab.py` — static
  `index.html` inspection. `TestCdn` (TC-047), `TestTabOrder` (TC-048
  five-tab order), `TestStatusColours` (TC-049 stylesheet kind/status
  pairs), `TestNoWebsocket` (TC-050 in both `index.html` and
  `server.py`), `TestTooltipsFromBackend` (TC-051 `renderPipeline` body
  has no hardcoded tooltip literals and references
  `explanations[...]`), `TestDragOneShot` (TC-055 static pattern check
  for 500 ms double-submit guard), `TestRollback` (TC-056 static check
  for `cy.remove` on 4xx branch).
- `.agent/adventures/ADV-009/tests/_fixtures.py` — `repo_root`,
  `ensure_on_syspath`, `TempAdventure` (context manager that writes a
  minimal `manifest.md` + `tasks/T1.md`, `tasks/T2.md` into
  `tempfile.mkdtemp()`; `shutil.rmtree(..., ignore_errors=True)` on
  exit), `ServerProcess` (launches `ThreadingHTTPServer` on
  `('127.0.0.1', 0)` in a daemon thread, exposes `.base_url`).

### Implementation Steps

1. Create `_fixtures.py` with the three helpers above. Not prefixed
   `test_`; `unittest discover` will skip it.
2. Create `test_ir.py`:
   a. Import-path setup via `_fixtures.ensure_on_syspath()` at module
      top.
   b. `TestSpecShapes` — three tests reading `.ark` files with
      `Path.read_text()` and regex-asserting declarations.
   c. `TestRoundTrip` — three tests calling `extract_ir("ADV-007")` /
      `extract_ir("ADV-008")`, asserting non-empty `tasks`, `documents`,
      `tcs`, `permissions`; `test_task_ids_match_manifest` parses the
      manifest `tasks:` frontmatter array by regex and asserts
      set-equality with IR task ids.
   d. `TestIrEntityShape` — `json.dumps(asdict(ir))` round-trips
      without error; enum-valued fields emit strings.
3. Create `test_graph_endpoint.py`:
   a. `setUpClass` on each class boots `ServerProcess` either against
      live ADV-007 (read-only tests) or against a `TempAdventure`
      (mutation tests).
   b. `TestImport.test_server_imports_ir` — opens `server.py`,
      asserts `"adventure_pipeline.tools.ir"` substring present.
   c. `TestStdlibOnly.test_no_third_party_imports` — `ast.parse` walk,
      assert every top-level module in `sys.stdlib_module_names | {"adventure_pipeline"}`.
   d. `TestGraphShape.test_top_level_keys` and `test_schema` — GET
      `/api/adventures/ADV-007/graph`, assert 200 + JSON shape.
   e. `TestDependsOn` — 4 HTTP cases: happy, self-cycle, transitive
      cycle, unknown task. Each mutation case re-reads target task file
      from disk to confirm frontmatter state.
   f. `TestCycleFree.test_self_and_transitive` — import `_cycle_free`
      directly from the `server` module; feed in-memory IR stub with a
      3-task chain.
4. Create `test_pipeline_tab.py`:
   a. `setUpModule` reads `index.html` once into a module-level
      string; `html.parser.HTMLParser` produces tag/attr lists.
   b. `TestCdn` — any `<script>` tag with `src` matching
      `r"https?://[^"']*cytoscape[^"']*\.js"`; no `./vendor/cytoscape`.
   c. `TestTabOrder` — parse `data-testid="tab-*"` hooks, assert
      `["tab-overview","tab-tasks","tab-documents","tab-pipeline","tab-decisions"]`.
   d. `TestStatusColours` — regex-assert kind/status pairs
      (task/--accent, tc/--muted|--ok|--danger, decision/--warn,
      document/--ink) each appear at least once.
   e. `TestNoWebsocket` — zero matches for `WebSocket`, `EventSource`,
      `wss?://` in both `index.html` and `server.py`.
   f. `TestTooltipsFromBackend` — extract `renderPipeline` body via a
      bracket-balancing helper in `_fixtures.py`; assert no literal
      string matches tooltip-like regex; assert `explanations\[` or
      `explanations\.` appears ≥ 1 time.
   g. `TestDragOneShot` — static pattern check: drag handler body
      contains a 500 ms debounce guard (disabled flag OR `setTimeout(...,
      500)`).
   h. `TestRollback` — `depends_on` POST error branch calls
      `cy.remove(` or equivalent.
5. Run `python -m unittest discover -s .agent/adventures/ADV-009/tests
   -p "test_*.py"` from repo root; expect exit 0 (after T015–T020
   land).
6. Spot-check each file individually: `python -m unittest
   .agent.adventures.ADV-009.tests.test_ir`, etc.
7. Manually scan test sources: every TC in
   `target_conditions` (minus TC-045, TC-057) maps to a declared
   `test_` function per the design's TC mapping table.

### Risks

- Authoring ordering: tests assume T015–T020 have landed; `depends_on`
  already enforces this.
- `renderPipeline` detection relies on the function being named
  literally `renderPipeline(adv)` (contract set by design-pipeline-graph-view).
- Ark spec tests use regex rather than the Ark parser; parse-cleanliness
  for TC-039 remains covered end-to-end by its manifest `proof_command`.
- Python 3.10+ required (`sys.stdlib_module_names`); manifest confirms
  Python 3.12 in the environment block.
