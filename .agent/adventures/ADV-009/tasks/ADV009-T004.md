## Description
Implement a new handler that returns a unified list of designs/plans/research/reviews with per-type metadata (`one_liner`, `task_count`, `waves`, `status`, `task_id`). Route it in `do_GET`. Stdlib-only.

## Acceptance Criteria
- [ ] `GET /api/adventures/{id}/documents` returns JSON array of `DocumentEntry`.
- [ ] A plan file with `## Wave 1` and `## Wave 2` headings reports `waves: 2`.
- [ ] Designs include a `one_liner` parsed from the first sentence of `## Overview` (<=120 chars).
- [ ] No new third-party imports.

## Design

Adds one new route `GET /api/adventures/{id}/documents` to
`.agent/adventure-console/server.py`. Implementation is additive:

- Four stdlib-only parser helpers (`_first_heading`, `_design_one_liner`,
  `_plan_metadata`, `_review_metadata`) defined at module scope for
  direct unit testing.
- One assembler `list_documents(adv_id)` that walks `designs/`, `plans/`,
  `research/`, and `reviews/` under the adventure directory, reading
  each markdown file and emitting a typed `DocumentEntry` record
  (designs → `one_liner`; plans → `task_count` + `waves`; research →
  `title`; reviews → frontmatter fields matching the existing
  `get_adventure` review shape). `adventure-report.md` is skipped from
  the reviews group to avoid double-surfacing.
- One routing branch added to `do_GET` immediately after the existing
  `/api/adventures/{id}` match, reusing the existing FileNotFoundError
  handler for missing adventures.

`waves` counts `^## Wave ` and `^## Phase ` headings. `task_count`
counts `^### ` headings under the first `^## Tasks` block.
`one_liner` is the first sentence of the `## Overview` section,
trimmed and capped at 120 chars.

No new imports — pure `re` + `pathlib` + existing helpers
(`parse_frontmatter`, `list_dir`, `read_text`, `adventure_path`).

(supersedes the shared `.agent/adventures/ADV-009/designs/design-backend-endpoints.md`
for the implementation-step breakdown specific to this task).
