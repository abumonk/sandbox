## Description
Render concept synopsis (first paragraph + Show more), TC progress bar with non-passing rows preview, and the next-action card driven by `summary.next_action`. Use `.card`, `.progress`, and `.disclosure` primitives. Add `data-testid` hooks.

## Acceptance Criteria
- [ ] Progress bar renders as a `<div class="progress">` with `data-testid="tc-progress-bar"` (not a 9-column table). (TC-018)
- [ ] Up to 5 non-passing TCs appear above the Show-all disclosure, sorted failed-before-pending, capped at 5. (TC-019)
- [ ] A `data-testid="next-action-card"` card exists; its title and button label differ by `a.state` per the six-row mapping in the design (concept / planning / review / active / blocked / completed). (TC-020)
- [ ] Raw concept markdown beyond the first paragraph is hidden behind `<details data-testid="show-details-concept">` labelled "Show more". (TC-021)
- [ ] Every top-level Overview block uses `class="card"` — no ad-hoc inline card chrome. (TC-033)
- [ ] `renderOverview(a)` works with `a.summary` absent (client-side `deriveSummary` / `deriveNextAction` fallbacks keep the tab renderable ahead of the T003 backend field).
- [ ] The legacy "Adventure Report" button at the bottom of the current `renderOverview` is removed (superseded by the `open_report` next-action kind).

## Design

See `.agent/designs/ADV009-T007-design.md` for the full implementation design.

**Summary.** Rewrite `renderOverview(a)` in `.agent/adventure-console/index.html` (currently lines 449–483) as three stacked `.card`s inside a `<div class="stack">`: (1) a concept synopsis card that prints the first paragraph of `a.concept` inline and hides the remainder behind `<details class="disclosure" data-testid="show-details-concept">`, (2) a TC progress card with a `.progress` bar plus a per-status count label, up to 5 non-passing TC preview rows (failed before pending), and a `<details data-testid="show-details-tcs">` disclosure wrapping the v1 full TC table (5 columns), and (3) a next-action card keyed by `a.summary.next_action.kind` (or a `deriveNextAction(a)` fallback off `a.state`) whose `data-testid="next-action-button"` onclick routes via `switchTab` to the appropriate v2 tab (Decisions / Tasks / Documents) or POSTs `/api/adventures/{id}/state` for `promote_concept`. Data-testid hooks per `design-test-strategy.md`: `tc-progress-bar`, `tc-progress-label`, `next-action-card`, `next-action-button`, `show-details-concept`, `show-details-tcs`. No backend calls — all data comes from `this.currentAdv`. No new CSS classes beyond T005 primitives (only two inline tweaks for `.tc-progress-label` and `.tc-row` if T005 didn't already declare them). The legacy "Adventure Report" button block is removed. The design doc enumerates the six-row next-action state mapping verbatim from `designs/design-overview-tab.md` and the five TC self-review checks required before flipping to review.
