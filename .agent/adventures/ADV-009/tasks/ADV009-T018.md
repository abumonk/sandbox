## Design

Detailed plan: `.agent/designs/ADV009-T018-design.md`. Summary:

Add two route branches to `.agent/adventure-console/server.py` plus a small
helper trio. All stdlib; the IR extractor from ADV009-T016 is the only
non-stdlib import (sibling package, made importable via a defensive
`sys.path.insert(0, REPO_ROOT)` block at module top).

Concrete steps:

1. Module top â€” insert `sys.path` shim and
   `from adventure_pipeline.tools.ir import extract_ir`. Update the
   docstring's *Endpoints* table.
2. Add helper `_load_ir(adv_id)` â€” wraps `extract_ir`; preserves
   `FileNotFoundError`, repackages other failures with an `ir_extract:`
   prefix.
3. Add helper `_cycle_free(ir, task_id, new_dep)` â€” adjacency dict from
   `ir.tasks.depends_on`, BFS from `new_dep`; returns False iff
   `task_id` is reachable. Self-loop case folded in naturally.
4. Add helper `_rewrite_task_depends_on(adv_id, task_id, merged)` â€”
   reads the task file, rewrites the single `depends_on:` line (inline
   list form) and the `updated:` line, preserves all other frontmatter
   and body. Refuses multi-line `depends_on:` block form with
   `ValueError` to surface schema drift.
5. Add `graph_payload(adv_id)` returning the `{nodes, edges,
   explanations}` shape from design-pipeline-graph-view. Node kinds:
   adventure, phase, task, document, tc, decision. Edge kinds:
   depends_on, satisfies, state_transition, addresses. Explanations
   strings derived deterministically from IR fields.
6. Add `add_task_dependency(adv_id, task_id, body, actor)`:
   regex-validate ids, reject self-cycle and unknown tasks (400),
   call `_cycle_free` (400 on cycle), dedup-merge, persist via
   `_rewrite_task_depends_on`, append audit line via `append_log`,
   return `{ok, task_id, depends_on}`.
7. Wire `do_GET`: add
   `re.fullmatch(r"/api/adventures/(ADV-\d{3})/graph", path)` branch
   serving `graph_payload`.
8. Wire `do_POST`: add
   `re.fullmatch(r"/api/adventures/(ADV-\d{3})/tasks/(ADV\d{3}-T\d{3})/depends_on", path)`
   branch invoking `add_task_dependency`. Existing
   `ValueError â†’ 400` / `FileNotFoundError â†’ 404` mappings handle all
   error cases; no new exception classes.

No edits to existing handlers (`update_state`, `approve_permissions`,
`approve_design`, `record_knowledge_selection`, `/api/file`).
