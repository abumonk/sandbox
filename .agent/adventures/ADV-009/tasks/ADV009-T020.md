## Description
Wire interactive editing onto the Pipeline tab:
- Click state-transition edges → POST state.
- Drag task→task → POST `depends_on` with optimistic edge placement +
  rollback on 4xx.
- Right-click → DOM context menu with the item set from
  `design-graph-edit-affordances.md` (open doc, approve design, approve
  permissions, apply knowledge suggestion, open task detail).
- Toasts for errors.
- 500 ms double-submit guard.

## Acceptance Criteria
- [ ] Drag gesture fires exactly one POST per completed gesture.
- [ ] On 4xx, the optimistic edge is removed and a `.toast` shows the server's error message.
- [ ] Context menu items route only to existing endpoints + the new `depends_on` POST; no other new routes introduced here.
- [ ] State-transition click posts to `/api/adventures/{id}/state` with the same payload as the v1 Decisions tab.

### Summary of concrete steps

1. Add `API.addDepends(id, taskId, depTaskId)` helper that POSTs to
   `/api/adventures/{id}/tasks/{taskId}/depends_on` (the endpoint
   landed by T018).
2. Introduce a shared `PipelineEdit._busy` / `guardOnce()` double-submit
   guard (500 ms window) wrapping every mutation gesture.
3. State-transition edge click (`cy.on('tap', 'edge[kind="state_transition"]')`)
   → parse target id `state:<name>` → `API.setState(adv.id, name)`;
   toast on success/error; no optimistic change (next poll tick
   re-renders).
4. Drag task A → task B: `taphold` starts a tentative ghost edge;
   `mousemove` on the canvas container tracks the hovered target node;
   `mouseup` finalises — if target is a task node B ≠ A, add an
   optimistic edge `{id:_opt-A-B, optimistic:true}` and POST via
   `API.addDepends`. On 4xx/5xx: `cy.remove('#_opt-A-B')` + error toast.
   On success: leave the optimistic edge in place for next poll tick to
   reconcile. Single-fire guaranteed by `guardOnce` + mouseup handler
   self-removal.
5. Right-click DOM context menu: add
   `<div id="pipeline-ctx" class="card">`, populate items from
   `node.data('kind')` per the affordance matrix (open document,
   approve design, approve permissions, apply knowledge suggestion,
   open task detail). Every mutation item routes through an
   `API.*` helper; no raw `fetch` / `/api/...` strings inside
   `PipelineEdit`. Hide menu on outside click, `Escape`, or
   cytoscape `tap`.
6. No tooltip work here — T019 already wires `explanations[id]`.
7. Attachment point: single line `PipelineEdit.attach(cy, adv)` at the
   tail of `renderPipeline(adv)` (added by T019). Merge is additive.
