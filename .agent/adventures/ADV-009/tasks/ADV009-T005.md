## Description
Add the new CSS tokens and component classes from design-visual-system (`.card`, `.pill`, `.progress`, `.chip-bar`, `.chip`, `.stack`, `.disclosure`) to `index.html`'s existing `<style>` block. No external CSS files. Keep all legacy class rules in place (they will be removed in T011 after the rewrite is wired up).

## Acceptance Criteria
- [ ] All seven new classes (`.card`, `.pill`, `.progress`, `.chip-bar`, `.chip`, `.stack`, `details.disclosure`) exist in the `<style>` block.
- [ ] No new `<link rel="stylesheet">` or external font imports are added (only the existing `marked.js` CDN `<script>` remains).
- [ ] Existing v1 styles still resolve (no v1 selector removed; browser smoke confirms no visual change to current UI).
- [ ] Seven new design tokens (`--card-bg`, `--card-border`, `--pill-bg`, `--chip-bg`, `--chip-active`, `--progress-bg`, `--progress-fill`) are added to the existing `:root` block.

## Design

Purely additive CSS patch to `index.html`'s single `<style>` block. Two
edits:

1. **Extend `:root`** (after `--badge-bg` on line 22) with seven new
   tokens: `--card-bg`, `--card-border`, `--pill-bg`, `--chip-bg`,
   `--chip-active`, `--progress-bg`, `--progress-fill`. Values taken
   verbatim from `design-visual-system.md`.
2. **Append a new `/* ===== v2 visual system (ADV-009) ===== */` section
   immediately before `</style>`** (currently line 228) containing the
   seven primitives: `.card` (+ `.card-title`, `.card-meta`), `.pill`
   (with `state-*` / `status-*` color variants mirroring `.badge`),
   `.progress` (`<div class="progress"><span style="width:N%"></span>
   </div>` pattern, 8px track), `.chip-bar` + `.chip` (keyboard-focus
   ring, `.chip.active` state), `.stack` (vertical flex, 16px gap), and
   `details.disclosure` (styled chevron via border-triangle, rotates on
   `[open]`; WebKit marker hidden).

No v1 class is edited or removed — `.split`, `.file-list`, `.concept-box`,
`.log-tail`, `.badge`, etc. remain intact. The new rules sit after all
v1 rules, so position-based cascade resolves any future `.pill`/`.badge`
collision in favor of the v2 primitive. Dead-CSS cleanup is deferred to
T011.

No HTML markup changes, no JS changes, no tests added — the visual
system becomes available as a stylesheet API for T006–T010 to consume.
Structural verification of the new classes (TC-031) and the "no external
stylesheet" invariant (TC-032) is handled by `test_ui_layout.py` per
T012.

See full design (token values, exact CSS bodies, implementation step
order, grep-based acceptance-check commands, and risk register) at
`.agent/designs/ADV009-T005-design.md`.
