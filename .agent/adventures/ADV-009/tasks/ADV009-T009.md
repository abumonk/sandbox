## Description
Build `renderDocuments` consuming `/api/adventures/{id}/documents`. Render chip bar + unified list. Implement per-type detail renderers: `renderDesignDoc` (one-line header from Overview + prose + TC/target-file side boxes + Approve button), `renderPlanDoc` (wave-grouped task cards), `renderResearchDoc` (prose + tag), `renderReviewDoc` (status strip + summary key-value + issues list + Show-full disclosure).

## Acceptance Criteria
- [ ] Chip bar renders with All/Designs/Plans/Research/Reviews chips.
- [ ] Chip click filters client-side (no network).
- [ ] Design opened -> shows "What this design decides:" header.
- [ ] Plan with `## Wave N` headings -> renders waves as visual groups.
- [ ] Review -> shows PASSED/FAILED badge and summary strip.

## Design

Implementation plan lives in
`.agent/designs/ADV009-T009-design.md` (concrete steps), driven by the
adventure-level design `.agent/adventures/ADV-009/designs/design-document-layouts.md`
and the backend contract in
`.agent/adventures/ADV-009/designs/design-backend-endpoints.md`.

Summary: add `renderDocuments(pane, a)` to
`.agent/adventure-console/index.html` as the handler for the new
`documents` tab (introduced in ADV009-T006). It fetches
`/api/adventures/{id}/documents` (shipped by ADV009-T004), renders a
`.chip-bar` with five chips (All / Designs / Plans / Research / Reviews)
that filter a unified row list client-side — no network on chip click.
Each row opens a per-type detail renderer:
- `renderDesignDoc` — "What this design decides:" one-liner parsed from
  `## Overview`, main prose with Overview/TC/Target-Files sections
  stripped, side cards for Target Conditions + Target Files, compact
  Approve button reusing `API.approveDes`.
- `renderPlanDoc` — parses `## Wave N` / `## Phase X` and groups `### `
  task cards into `.wave-group` bands; falls back to plain markdown.
- `renderResearchDoc` — small `research` tag + rendered prose.
- `renderReviewDoc` — top status strip with PASSED/FAILED pill, compact
  `## Summary` key-value strip, severity-coded issues list, and a
  `<details class="disclosure">` with the full review body.

Shared helpers (`splitSections`, `parseSummaryTable`, `parseIssuesList`)
land next to existing `parseKnowledgeSuggestions`. All styling uses
visual primitives from ADV009-T005 (`.card`, `.pill`, `.chip-bar`,
`.chip`, `.stack`, `.disclosure`); four new color variants
(`.pill.type-{design,plan,research,review}`) and a handful of layout
rules are added inside the existing `<style>` block. No new external
files, no backend changes.

Dispatcher wiring: add a `tab === 'documents'` branch in `renderTab()`
before the legacy design/plan/research/reviews branches. Document
results are cached on `Console.documentsCache`, invalidated on
`reopen()` and adventure switch. Legacy `renderFileBrowser` and
`renderReviews` remain until ADV009-T011 retires them.

Testability hooks: `data-testid="documents-chips"`,
`data-testid="chip-{kind}"`, `data-testid="documents-list"`,
`.wave-group`, `.review-strip`, and `.design-header` anchor the five
acceptance-criteria checks and TC-013..TC-017 from
`design-test-strategy`.
