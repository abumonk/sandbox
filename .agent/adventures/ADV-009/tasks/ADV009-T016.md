## Design

**Summary**: Author a stdlib-only Python IR extractor under
`adventure_pipeline/tools/` (sibling package scaffolded in T015). The
extractor reads a live `.agent/adventures/ADV-NNN/` directory — manifest
frontmatter + TC markdown table + `designs/`, `plans/`, `research/`,
`reviews/`, `tasks/`, `permissions.md` — and returns an
`AdventurePipelineIR` dataclass mirroring the Ark entity shapes declared
in `specs/adventure.ark`. Parsing reuses the regex-driven patterns from
`.agent/adventure-console/server.py` (`parse_frontmatter`,
`_target_conditions`, `get_adventure`); no PyYAML, no third-party deps.
A CLI `python -m adventure_pipeline.tools.ir ADV-NNN` prints
`to_json(ir)` to stdout.

**Key IR shape**: `AdventurePipelineIR { id, title, state, created,
updated, concept, tasks_list (from manifest frontmatter), tasks[],
documents[] (with kind: design|plan|research|review), tcs[],
decisions[] (empty for ADV-007/008, reserved for ADV-009+),
permissions[] (category: shell|file|mcp|external), agents[], roles[],
log_tail }`.

**Pattern precedent**: ADV-008 `shape_grammar/tools/ir.py` — same
sibling-package-with-IR-extractor pattern, stdlib only.

See `.agent/designs/ADV009-T016-design.md` for full dataclass
declarations, per-section parser design, permissions-table walker,
CLI layout, and testing strategy.

**Source design (domain)**: `.agent/adventures/ADV-009/designs/design-ark-pipeline-spec.md` §4.
