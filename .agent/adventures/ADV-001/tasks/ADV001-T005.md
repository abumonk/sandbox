## Design

### Summary

Extend `ark_grammar.lark` with 6 changes to mirror pest grammar's expression/predicate support:

1. **`item` rule** (line 12-21): Add `| expression_def` and `| predicate_def`
2. **New section after `verify_def`** (after line 121): Add `expression_def`, `predicate_def`, `expression_body`, `predicate_body` rules
3. **`?expr` rule** (line 158): Change `?expr: or_expr` to `?expr: pipe_expr`
4. **New rules after `?expr`**: Add `pipe_expr: or_expr ("|>" pipe_stage)*`, `pipe_stage`, `pipe_fn_ident: IDENT ("-" IDENT)*`
5. **`?atom` rule** (lines 170-181): Insert `var_ref`, `prop_ref`, `idx_ref`, `nested_ref` before `fn_call`
6. **New param-ref rules**: `var_ref: "@" IDENT`, `prop_ref: "[" IDENT ("." IDENT)+ "]"`, `idx_ref: "#" IDENT "[" NUMBER "]"`, `nested_ref: "{" expr "}"`

Key Lark-specific decisions:
- `pipe_fn_ident` is a **rule** (not terminal) to scope kebab-case to pipe stages only
- `pipe_expr` uses `?` prefix so it's transparent when no `|>` is present (single-child passthrough); when a pipe IS used, the node materializes for the transformer
- `prop_ref` requires dotted path (`IDENT ("." IDENT)+`) to avoid collision with `array_expr`
- `nested_ref` uses `{expr}` safely since no other `{`-prefixed atom exists
