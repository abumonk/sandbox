### Summary

Add 3 dataclasses (`ExpressionDef`, `PredicateDef`, `PipeStage`) and 10 transformer methods to `ark_parser.py` to handle the Lark grammar nodes added by T005.

**Dataclasses** (after `EnumDef`, before `ArkFile`):
- `ExpressionDef(kind, name, inputs, output, chain, description)` -- top-level expression item
- `PredicateDef(kind, name, inputs, check, description)` -- top-level predicate item
- `PipeStage(name, args)` -- sub-node of pipe expressions

**Transformer methods** (10 new):
- `pipe_expr` -- passthrough if 1 child, else `{"expr": "pipe", "head": ..., "stages": [...]}`
- `pipe_stage` -- returns `PipeStage(name, args)`
- `pipe_fn_ident` -- joins IDENT parts with hyphens for kebab-case
- `var_ref` -- `{"expr": "param_ref", "ref_kind": "var", "name": ...}`
- `prop_ref` -- `{"expr": "param_ref", "ref_kind": "prop", "parts": [...]}`
- `idx_ref` -- `{"expr": "param_ref", "ref_kind": "idx", "name": ..., "index": ...}`
- `nested_ref` -- `{"expr": "param_ref", "ref_kind": "nested", "inner": ...}`
- `expression_body` -- intermediate: returns dict of inputs/output/chain
- `expression_def` -- returns `ExpressionDef` dataclass
- `predicate_body`, `predicate_def` -- analogous for predicates

**Existing method changes**:
- `start()` -- add `ExpressionDef, PredicateDef` to isinstance tuple
- `ArkFile` -- add `expression_index`, `predicate_index` dict fields
- `_build_indices()` -- populate expression_index/predicate_index via enumerate

**Import resolution**: No changes needed -- existing generic resolver handles new item types automatically.

**JSON serialization**: PipeStage converted to plain dict inline in `pipe_expr` to avoid serialization issues.
