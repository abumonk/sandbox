### Summary

Edit `ark.pest` (239 lines) to add 10 new rules for pipe expressions and expression/predicate items:

**Precedence chain change** (line 145-146):
- `expression` now wraps `pipe_expr` instead of `or_expr` directly
- `pipe_expr = { or_expr ~ ("|>" ~ pipe_stage)* }` -- lowest-precedence binary operator
- `pipe_stage = { pipe_fn_ident ~ call_tail? }` -- stage is a function name + optional args
- `pipe_fn_ident = @{ ... }` -- atomic rule allowing kebab-case (e.g., `text-to-lower`), only valid inside pipe stages

**Parameter references** (add to `atom` rule, before `path_call_expr`):
- `var_ref = ${ "@" ~ ident }` -- compound-atomic, no whitespace after `@`
- `prop_ref = { "[" ~ ident ~ ("." ~ ident)+ ~ "]" }` -- dotted only, single-ident stays `array_expr`
- `idx_ref = ${ "#" ~ ident ~ "[" ~ number ~ "]" }` -- compound-atomic
- `nested_ref = { "{" ~ expression ~ "}" }` -- recursive

**Top-level items** (add to `item` rule before `verify_def`):
- `expression_def = { "expression" ~ ident ~ "{" ~ expression_body ~ "}" }`
- `predicate_def = { "predicate" ~ ident ~ "{" ~ predicate_body ~ "}" }`
- `expression_body = { "in:" ~ typed_field_list ~ "out:" ~ type_expr ~ "chain:" ~ expression }`
- `predicate_body = { "in:" ~ typed_field_list ~ "check:" ~ expression }`

**Ordering in `atom`**: `prop_ref` before `array_expr` to prevent `[a.b.c]` from being consumed as an array. `var_ref`, `idx_ref`, `nested_ref` before `path_call_expr`.

**Collision analysis**: No conflicts with existing `@in`/`@out`, `#process`, `[array]`, or `{enum_constraint}` -- all resolved by parse context (see design doc for full table).
