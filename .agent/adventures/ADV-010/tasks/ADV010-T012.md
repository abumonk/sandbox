## Description
Implement `.agent/telemetry/tools/backfill.py` — the top-level CLI. Merge candidates per task, pick highest-confidence field-by-field, emit MetricsRow set. Write to `metrics.md.new`; emit unified diff; if `--apply`, rename with backup. Covers `--adventure ADV-NNN`, `--all`, `--sources ...`, `--apply`, `--dry-run`.

## Acceptance Criteria
- `python -m telemetry.tools.backfill --adventure ADV-008 --dry-run` emits a diff without modifying `ADV-008/metrics.md`.
- `--apply` renames original to `metrics.md.backup.<ts>`.
- Running twice without `--apply` produces identical diffs (idempotency).
- Row's Run ID is stable across runs.
- Unreconstructable task → row with `result: unrecoverable`.

## Design

**Approach**: Single-file CLI module `.agent/telemetry/tools/backfill.py` that collects candidates from T011's reconstructors, merges per-task using field-level confidence ranking, writes `metrics.md.new`, emits unified diff, and optionally applies via atomic rename with timestamped backup.

**Design doc**: [ADV010-T012-design.md](../designs/ADV010-T012-design.md)

**Key implementation steps** (12 total):
1. Import from sibling modules: `reconstructors.*`, `schema`, `cost_model`, `aggregator`
2. Adventure discovery (`--adventure ADV-NNN` or `--all`)
3. Candidate collection — call each enabled reconstructor, group by task_id
4. Per-task merge — for each MetricsRow field, pick highest-confidence candidate value; ties broken by source priority (existing > log > task_log > git); never emit `confidence: high`
5. Unreconstructable tasks — emit row with `result: unrecoverable`, `confidence: estimated`, zero tokens
6. Full adventure reconstruction — discover tasks via glob, merge, sort by timestamp
7. Write `metrics.md.new` with YAML frontmatter + pipe-table (reuses aggregator totals logic)
8. Generate unified diff via `difflib.unified_diff`
9. Apply with backup — `os.replace()` for atomic rename; backup named `metrics.md.backup.<ts>`
10. CLI parser — `argparse` with `--adventure`, `--all`, `--apply`, `--dry-run`, `--sources`
11. Main entry point with summary output (adventures processed, rows, unrecoverable count)
12. Idempotency via deterministic Run ID = `sha1(adventure_id|agent|task|model|timestamp|)[:12]`

**Reversibility contract**: Without `--apply`, original `metrics.md` is never touched. With `--apply`, original is renamed to `metrics.md.backup.<ts>` before the new file takes its place. Rollback = `mv backup original`.
