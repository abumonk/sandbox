## Description
In `.agent/telemetry/capture.py`'s `main()`, after `recompute_frontmatter`, if `event.task` is non-empty and `event.result` is terminal, call `task_actuals.update(manifest_path(event.adventure_id), event.task)`. Error-isolate the call.

## Acceptance Criteria
- End-to-end test: synthetic event with `task: ADV010-T005` → manifest's T005 row gains Actual Duration/Tokens/Cost/Variance.
- `TaskActualsError` in the call does not raise into `main()`.

## Design

**Approach**: Add a post-aggregation conditional call to `task_actuals.update()` in `capture.py`'s `main()`, guarded by `event.task` being non-empty and `event.result` being terminal. The call sits inside the existing `except Exception` guardrail from `design-error-isolation.md`, so `TaskActualsError` is caught, logged, and exits 0.

**Key steps**:
1. Import `task_actuals` module (from T009)
2. Define `manifest_path(adventure_id)` helper returning `.agent/adventures/{ADV-ID}/manifest.md`
3. Define `TERMINAL_RESULTS = frozenset({"done","complete","passed","ready","failed","error"})`
4. After `recompute_frontmatter()` in `main()`, add: `if event.task and event.result in TERMINAL_RESULTS: task_actuals.update(manifest_path(event.adventure_id), event.task)`
5. No additional try/except needed -- outer guardrail covers it

Full design: [ADV010-T010-design.md](../designs/ADV010-T010-design.md)
