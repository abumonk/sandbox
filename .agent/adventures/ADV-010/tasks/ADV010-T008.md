## Description
Extend `.claude/settings.local.json` with `hooks.SubagentStop` and `hooks.PostToolUse` blocks per `design-hook-integration.md`, preserving the existing 128-entry `permissions.allow` list byte-for-byte. Write a small one-shot merger script (run locally by the implementer, not shipped) so the merge is reviewable.

## Acceptance Criteria
- `jq '.hooks.SubagentStop | length'` returns `>= 1`.
- `jq '.hooks.PostToolUse[0].matcher'` returns `"Task"`.
- Byte diff on `.permissions.allow` is empty.
- Autotest parses the file and asserts both hooks point at `python .agent/telemetry/capture.py`.

## Design

**Approach**: Insert a `hooks` top-level key into `.claude/settings.local.json` with two hook arrays (`SubagentStop` and `PostToolUse`) per `design-hook-integration.md`. The existing `permissions.allow` list must remain byte-for-byte identical. Prefer the Edit tool (surgical insertion before the closing brace) over a full JSON round-trip to avoid formatting drift.

**Implementation steps**:
1. Read `.claude/settings.local.json` and snapshot `.permissions.allow` via `jq` for later diff.
2. Use the Edit tool to replace the final `}` closing of the `permissions` block + file-level `}` with a comma-separated continuation adding the `hooks` block (SubagentStop matcher=`*`, PostToolUse matcher=`Task`, both pointing at `python .agent/telemetry/capture.py`, timeout 5000ms).
3. Verify with `jq` assertions: SubagentStop length >= 1, PostToolUse matcher = "Task", both commands contain `capture.py`.
4. Diff `.permissions.allow` before/after to prove zero clobber.

**Full design**: [ADV010-T008-design.md](../designs/ADV010-T008-design.md)
