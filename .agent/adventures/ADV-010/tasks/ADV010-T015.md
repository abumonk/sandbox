## Description
Author all test fixtures under `tests/fixtures/`: 7 event JSON files (happy_opus, happy_sonnet, missing_tokens, bad_model, replay, post_tool_use, malformed.txt); 5 metrics.md files; 2 manifest files.

## Acceptance Criteria
- All event JSON files validate or fail-validate as documented (self-check: running `schema.validate_event` on each produces the expected outcome).
- Every metrics fixture has a known-good frontmatter-totals pair computable by hand.

## Design

Author 14 fixture files across 3 directories under `tests/fixtures/`. Event fixtures (7 files) cover valid payloads for opus and sonnet models, a PostToolUse variant, and 3 invalid variants (missing tokens_in, unknown model, non-JSON text). Metrics fixtures (5 files) cover empty, single-row, multi-model aggregation, tilde-prefixed legacy values, and tampered-frontmatter healing. Manifest fixtures (2 files) cover actuals propagation with and without estimates for Variance computation. All numeric values are hand-computed from config.md rates (opus=0.015, sonnet=0.003, haiku=0.001) and documented in the design.

Full design: [`.agent/designs/ADV010-T015-design.md`](../../designs/ADV010-T015-design.md)

### Key implementation details

1. **Event fixtures**: 4 valid JSON payloads (happy_opus, happy_sonnet, post_tool_use, replay) matching SubagentEvent dataclass fields exactly. 2 invalid JSON payloads (missing_tokens, bad_model) that trigger specific PayloadError paths. 1 non-JSON file (malformed.txt) for stdin parse failure.
2. **replay.json**: Byte-identical copy of happy_opus.json to produce the same Run ID for idempotency testing (TC-CC-3).
3. **Metrics fixtures**: Use the new 12-column row schema (Run ID through Confidence), not the legacy 8-column format. Hand-verified frontmatter-to-row sums for each file.
4. **Manifest fixtures**: Use the 10-column ManifestEvaluationRow schema. with_estimates.md has Est. Tokens=45000 to enable Variance percentage computation.
5. **Cost values**: happy_opus cost = (85000+28000)/1000*0.015 = 1.6950; happy_sonnet cost = (20000+3500)/1000*0.003 = 0.0705.
