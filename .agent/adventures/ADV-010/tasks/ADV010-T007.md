## Description
Implement `.agent/telemetry/capture.py` with a `main()` guardrail that catches every exception, logs to `capture-errors.log`, and exits 0. Wire T004..T006. Implement `normalize_payload`, `adventure_id` resolution from cwd/env. Implement idempotent `append_row`.

## Acceptance Criteria
- Subprocess test: valid JSON on stdin → new row in correct metrics.md, exit 0.
- Subprocess test: malformed JSON → exit 0, one line in capture-errors.log.
- Subprocess test: same event twice → one row (idempotency).
- Subprocess test: `KeyboardInterrupt` propagates.

## Design

**Approach**: Implement `capture.py` as the hook entrypoint that wires T004 (cost_model), T005 (schema), and T006 (aggregator) behind a `main()` guardrail. Implement `log.py` for JSONL error logging. The guardrail catches all `Exception` subclasses, logs to `capture-errors.log`, and exits 0. `KeyboardInterrupt`/`SystemExit` propagate.

**Key functions in `capture.py`**:
- `read_stdin()` - reads and returns raw stdin; raises `PayloadError` on empty
- `normalize_payload(raw)` - aliases wire fields (`event`->`event_type`, `usage.input_tokens`->`tokens_in`, etc.)
- `resolve_adventure_id(payload)` - 5-step resolution: payload field, task_id prefix, env var, cwd parse, fail
- `append_row(adventure_id, row)` - idempotent append (checks Run ID collision before writing)
- `main(argv)` - the guardrail: read -> parse -> normalize -> validate -> build_row -> append -> aggregate -> task_actuals, all wrapped in `except Exception`

**Key functions in `log.py`**:
- `log_capture_error(exc, raw_payload)` - appends one JSONL line with `ts`, `exc`, `msg`, `payload_excerpt` to `capture-errors.log`; never raises

**Testing**: All 4 acceptance criteria verified via `subprocess.run()` against the real module (no mocks). Idempotency tested by piping the same event twice and asserting one row.
