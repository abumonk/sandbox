## Description
Implement `.agent/telemetry/task_actuals.py`: `update(manifest_path, task_id)`. Parses the `## Evaluations` table, finds the row for `task_id`, computes actuals from the adventure's `metrics.md`, rewrites that single row in place. All other bytes in the manifest preserved. Emits `Variance` against `Est. Tokens`.

## Acceptance Criteria
- Given fixture manifest with one task row and a metrics.md with a matching row, actuals columns get populated.
- Variance equals signed percentage vs Est. Tokens (hand-verified fixture).
- `diff` between pre- and post-update manifest shows exactly one changed row.
- No-op if manifest has no matching row (log line, no crash).
- Multiple rows for same task → actuals are sum.

## Design
Implement `.agent/telemetry/task_actuals.py` with one public function `update(manifest_path, task_id)` that: (1) reads metrics.md rows via `aggregator.parse_rows()`, (2) filters to rows matching `task_id`, (3) sums actuals (duration via `format_duration`, tokens as `tokens_in + tokens_out`, cost as sum of `cost_usd`), (4) computes variance as signed percentage of actual vs estimated tokens, (5) locates the matching row in the manifest's `## Evaluations` pipe-table by parsing the header dynamically, (6) rewrites only the four actuals cells in that row, (7) writes atomically via tmp+`os.replace()`. No-op with log if no matching row. All errors wrapped in `TaskActualsError`. Stdlib only.
