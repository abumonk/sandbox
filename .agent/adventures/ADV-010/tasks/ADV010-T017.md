## Description
Author a short operator doc at `.agent/telemetry/README.md` covering: layout, how the hook is wired, how to run backfill, how to read `capture-errors.log`, how to add a new cost rate. ≤ 200 lines.

## Acceptance Criteria
- Covers the 5 topics above as `## Sections`.
- Contains the discover one-liner.
- Contains the backfill one-liner.

## Design

Create `.agent/telemetry/README.md` (max 200 lines) with five `##` sections:

1. **Layout** - directory tree of `.agent/telemetry/` with one-line module descriptions (capture.py, schema.py, cost_model.py, aggregator.py, task_actuals.py, errors.py, tools/backfill.py, tools/reconstructors/, capture-errors.log)
2. **How the Hook is Wired** - hook in `.claude/settings.local.json`, fires on SubagentStop/PostToolUse, pipes JSON to stdin of `python .agent/telemetry/capture.py`
3. **How to Run Backfill** - one-liner `python -m telemetry.tools.backfill --adventure ADV-NNN`, explain confidence levels (high/medium/low), safety (writes .new then renames)
4. **How to Read `capture-errors.log`** - location, format, exit-0 guardrail, exception hierarchy, common triage patterns
5. **How to Add a New Cost Rate** - edit `.agent/config.md` frontmatter `token_cost_per_1k:` block, add `model: rate` line, no code changes needed

Must include the discover one-liner (`python -m unittest discover -s .agent/adventures/ADV-010/tests -v`) and the backfill one-liner.
