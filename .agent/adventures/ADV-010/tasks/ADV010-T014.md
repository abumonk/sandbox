## Description
With hook installed and capture.py wired, trigger one real subagent run bound to ADV-009 (a small research sub-task: e.g. "summarize ADV-009's current state in one paragraph"). Verify that `ADV-009/metrics.md` gains exactly one new row with `Confidence: high`, non-zero tokens, the model set correctly, and that frontmatter totals update to match.

## Acceptance Criteria
- New row present with `Confidence: high`.
- `total_tokens_in` after = `total_tokens_in` before + row.tokens_in.
- `agent_runs` incremented by exactly 1.
- `test_adv009_canary_row_populated` passes.

## Design

**Approach**: Snapshot ADV-009/metrics.md pre-state, invoke a real subagent via the Task tool with a minimal research prompt bound to ADV-009, then verify that the capture hook fired and produced exactly one new row with `Confidence: high`, non-zero tokens, correct model, and that frontmatter totals (total_tokens_in, agent_runs, etc.) incremented by exactly the new row's values. Record pre/post snapshots as fixtures for `test_live_canary.py`.

**Implementation steps**:
1. Snapshot `ADV-009/metrics.md` to `tests/fixtures/canary/pre_metrics.md`, recording frontmatter totals and row count.
2. Trigger one real subagent run via Task tool: a small research sub-task ("summarize ADV-009's current state in one paragraph") scoped to ADV-009.
3. Re-read `ADV-009/metrics.md`, confirm exactly one new row appended with: Confidence=high, tokens_in>0, tokens_out>0, valid model, 12-char hex Run ID, cost>0, duration>0.
4. Assert frontmatter: `total_tokens_in`(after) = `total_tokens_in`(before) + row.tokens_in; `agent_runs`(after) = `agent_runs`(before) + 1.
5. Write post-state to `tests/fixtures/canary/post_metrics.md`.
6. Run `test_adv009_canary_row_populated` and confirm it passes.

**Risks**: Hook not firing (verify settings.local.json first); adventure_id resolving to ADV-010 instead of ADV-009 (scope sub-task to ADV-009 task_id); double-fire from SubagentStop+PostToolUse (idempotency guard should handle).
