## Design

Operator task: run the backfill CLI (from T012) on each of ADV-001..ADV-009 using the telemetry-historian role's prescribed workflow. Five phases: (1) pre-flight dependency and state checks, (2) dry-run each adventure individually and review diffs for confidence correctness, tilde removal, and plausible token estimates, (3) apply per-adventure sequentially with backup verification after each, (4) run autotests (`test_every_completed_adventure_has_runs`, `test_backfill_rows_never_high_confidence`) and grep for residual tildes, (5) rollback protocol if any verification fails.

Key constraints from the telemetry-historian role: never apply without dry-run review first; never emit `Confidence: high` (reserved for live hook); backup files must be committed alongside rewritten metrics. ADV-008 requires special attention as it has 34 existing rows whose numeric values must be preserved (TC-BF-2).
