## Description
Implement `.agent/telemetry/schema.py`: the `SubagentEvent` dataclass, `MetricsRow` dataclass, `validate_event(dict) -> SubagentEvent`, `build_row(event, cost)`, row serialization/parsing for pipe-table format. Re-use `errors.py` from T004.

## Acceptance Criteria
- Every documented invalid payload variant raises a specific exception (10 cases).
- `build_row(event)` produces a row whose Run ID is the 12-hex SHA-1 prefix of the canonical key.
- Round-trip: `serialize(row)` then `parse_row(line)` returns byte-equivalent row for all numeric columns.

## Design

Implement `.agent/telemetry/schema.py` with two frozen dataclasses (`SubagentEvent`, `MetricsRow`), a strict `validate_event()` covering 10 rejection cases via `PayloadError`, a `build_row()` that derives a 12-hex SHA-1 Run ID from the canonical key, and round-trippable `serialize()`/`parse_row()` for pipe-table format. Depends on `errors.py` and `cost_model.py` from T004.

Full design: [`.agent/designs/ADV010-T005-design.md`](../../designs/ADV010-T005-design.md)

### Key implementation details

1. **validate_event(dict) -> SubagentEvent** - 10 named rejection cases (missing fields, type violations, regex mismatches, unknown model). Raises `PayloadError` with specific messages.
2. **build_row(event, cost) -> MetricsRow** - Run ID = `sha1("{adv}|{agent}|{task or '-'}|{model}|{ts}|{session or ''}")[:12]`. Duration clamped to `max(1, round(ms/1000))`. Confidence = `"high"`.
3. **serialize(row) -> str** - Pipe-separated line, cost formatted to 4dp.
4. **parse_row(line) -> MetricsRow** - Split on `|`, coerce int/float columns, raise `SchemaError` on mismatch.
