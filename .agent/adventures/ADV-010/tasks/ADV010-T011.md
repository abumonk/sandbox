## Description
Implement the 4 reconstructor submodules under `.agent/telemetry/tools/reconstructors/`:
- `existing_rows.py` (parse current metrics.md, strip tildes, confidence=medium)
- `log_parser.py` (parse adventure.log, extract spawn/complete pairs)
- `git_windows.py` (shell out to `git log --pretty=... --name-only`, derive per-task commit windows)
- `task_logs.py` (read task-file `## Log` sections)

Each returns `List[Candidate]`; a Candidate has the same shape as MetricsRow plus a `source` field.

## Acceptance Criteria
- `existing_rows.parse(ADV-008/metrics.md)` returns 34 candidates with tildes stripped and integer tokens.
- `log_parser.parse(ADV-008/adventure.log)` returns ≥ 19 spawn events (one per task).
- `git_windows.for_adventure("ADV-008")` returns per-task windows (using `git log` bounded by created..updated).
- Each reconstructor has at least one fixture-based unit test.

## Design
Implement four reconstructor submodules under `.agent/telemetry/tools/reconstructors/`, each extracting telemetry candidates from a different data source. All share a common `Candidate` dataclass (MetricsRow shape + `source` field) and return `List[Candidate]`. Stdlib-only (re, datetime, subprocess, pathlib, dataclasses).

- **`existing_rows.py`**: Parse `metrics.md` pipe-tables. Strip `~` from numeric fields, convert duration strings (`NNmin`, `NNs`) to integer seconds. Confidence=medium, source=existing.
- **`log_parser.py`**: Parse `adventure.log` lines via regex, pair spawn/complete events by agent, compute duration deltas. Confidence=low, source=log.
- **`git_windows.py`**: Shell out to `git log --pretty=format:"%H|%ai|%s" --name-only`, group commits by task ID, compute first-to-last commit windows. Confidence=low, source=git.
- **`task_logs.py`**: Glob task `*.md` files, extract `## Log` sections, parse timestamps for created-to-final-status deltas. Confidence=low, source=task_log.

Full design: [ADV010-T011-design.md](../designs/ADV010-T011-design.md)
